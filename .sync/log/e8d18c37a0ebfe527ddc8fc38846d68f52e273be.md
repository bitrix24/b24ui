# Port: fix(Select): open menu on label click (#6575)

**Upstream:** `e8d18c37a0ebfe527ddc8fc38846d68f52e273be` (nuxt/ui)
**Decision:** port (behavioral fix; theme `replaceFocus` removal skipped)

## Upstream change
1. `Select.vue`: add `onTriggerClick(open)` — a native `<label for>` click
   forwards a `click` with no `pointerdown`, so the menu stays closed; when
   `!open`, re-dispatch a `pointerdown` to open it (matching SelectMenu). Wire
   `@click="onTriggerClick(open)"` on the `SelectTrigger`.
2. `select.ts` / `select-menu.ts`: remove the `replaceFocus` compoundVariants
   transform (tied to upstream's separate focus-uniformization, #6576).
3. spec: a "label" test; regenerated Select snapshots (focus-class churn from #2).

## b24ui adaptation
- `Select.vue`: ported `onTriggerClick` verbatim + `@click="onTriggerClick(open)"`
  on `SelectTrigger` (`open` from the `SelectRoot` v-slot). 1:1.
- spec: added the "label" test (`B24FormField` + `B24Select`, `renderForm` /
  `slotVars`). +2 (nuxt/vue).
- **Theme `replaceFocus` removal — skipped (scoped deviation).** b24ui's
  transforms differ and are NOT the same model as upstream's:
  `select.ts`'s `replaceFocus` is **already a no-op** (body commented out) and
  is woven into b24ui's air-color `compoundVariants`; `select-menu.ts`'s is an
  active b24ui-specific `focus:` → `focus-visible:` swap. Removing them would
  alter b24ui focus styling and is tied to upstream's focus-uniformization
  (b026ca2c), which b24ui handles separately. No Select snapshot churn results
  (the `@click` adds no markup).
