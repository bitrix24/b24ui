# Port: fix(Tabs): render active indicator during SSR (#6570)

**Upstream:** `9e5b8a6bff4f9f12fc6282b450466ec68dc9017d` (nuxt/ui)
**Decision:** port (theme core; docs-app tweaks skipped)

## Upstream change
reka-ui only renders the measured `TabsIndicator` on the client, so SSR /
pre-hydration markup has no active-tab highlight. Fix in `src/theme/tabs.ts`:
add an `ssr(...)` helper that gates CSS-only pseudo-element fallback classes on
the active trigger by the **absence** of the indicator element
(`in-[[data-slot=list]:not(:has([data-slot=indicator]))]:data-[state=active]:…`),
applied across the pill/link variants, orientation compounds and color loops.
Plus regenerated snapshots and three docs-app hydration tweaks
(FrameworkTabs.vue, useNavigation.ts, [...slug].vue).

## b24ui adaptation
b24ui's Tabs theme is heavily diverged: a plain object (no `options` /
`theme.colors` loops), only a **`link`** variant (no `pill`), air tokens.
Ported the SSR fallback to b24ui's `link` variant:
- added the `ssr()` helper (verbatim).
- `link` variant trigger: `['focus:outline-none', ssr('after:content-[\'\']',
  'after:absolute', 'after:rounded-(--ui-border-radius-pill)')]`.
- horizontal.link compound: `trigger: ssr('after:inset-x-0',
  'after:-bottom-[calc(var(--spacing)+1px)]', 'after:h-px')`.
- vertical.link compound: `trigger: ssr('after:inset-y-0',
  'after:-start-[calc(var(--spacing)+1px)]', 'after:w-px')`.
- color compound (`variant: 'link'`): appended
  `ssr('after:bg-(--ui-color-design-selection-content)')` so the fallback line
  matches the real indicator color.
- regenerated Tabs snapshots (46 updated).

### Deviation — docs-app tweaks skipped
The three docs-app changes are nuxt/ui-docs-specific hydration fixes for the
framework switcher and are **not** part of the component theme:
- `docs/app/pages/docs/[...slug].vue` — **does not exist** in b24ui.
- `FrameworkTabs.vue` / `useNavigation.ts` exist but b24ui's framework-tab /
  prerender-hydration setup differs (the upstream class targets the `pill`
  style b24ui doesn't have); they work as-is. Skipped to avoid an unrelated,
  mismatched docs refactor. The theme-level SSR fallback covers all Tabs usage.
