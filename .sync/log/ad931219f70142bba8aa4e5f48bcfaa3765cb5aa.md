## Deviation — `@tiptap/pm` override (b24ui-specific)
b24ui's dep tree pulls `@tiptap/y-tiptap@3.0.3` (via `extension-collaboration`),
which pins `@tiptap/pm@^3.26.x` → a **second** `@tiptap/pm@3.26.1` +
`prosemirror-view@1.41.8` alongside the bumped `3.27.3`. Two `@tiptap/pm/view`
copies made `EditorCompletionExtension.ts`'s `decorations()` return
(`DecorationSet`) non-assignable to the plugin's expected `DecorationSource`
(TS2322) — a duplicate-types mismatch, not a real API change. Added
`'@tiptap/pm': ^3.27.1` to the existing `pnpm-workspace.yaml` `overrides` block
so all `@tiptap/pm` (incl. `y-tiptap`'s) dedupe to 3.27.3 → single
`@tiptap/pm/view` → types match. (A blanket `pnpm dedupe` was rejected: it
churned nuxt's install paths and triggered unrelated `TS2883` declaration-emit
errors in `src/runtime/plugins/*`; the targeted override is minimal.) The
example file itself is untouched.
